simple-www-logrotate

Rotates HTTP access log files.

Usage:

simple-www-logrotate [-v]
Option Description
-v Print verbose output.

Rotates HTTP access log files by renaming *.log files in a directory hierarchy. Immediately afterwards a command is run to create new log files.

The old log files are compressed with lzma and processed with simple-www-stats. After a configurable number of days, the old log files are removed.

Configuration:

The /etc/simple-www-logrotate.conf file contains a number of shell variables that controls the log file rotation:

# Shell command to release old log files
COMMAND="/usr/sbin/nginx -s reopen"

# Base directory for access log files
LOG_DIR=/var/log/www

# Maximum log file age (days)
LOG_EXPIRES=356

# Statistics directory
STAT_DIR=/srv/logstats
