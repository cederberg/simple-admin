admin-status

Checks and prints the machine and process status.

Usage:

admin-status [-s]
Option Description
-s Prints a short summary without listing process status.

Prints a summary of the most important machine and process metrics. Processes are checked for existence only, not probed in any other way. An example output:

loadavg:    0.00, 0.01, 0.05       -- 4 processors
memory:     298 MB (60.1%) free    -- 108 rss, 90 cache, 9 swap, 496 total (MB)
storage:    16.3 GB (69.4%) free   -- 7.2 GB used, 23.6 GB total on /dev/xvda
* cron:     OK                     -- pid:  2223,  rss:   0.8 MB,  swap:   0 MB
* ntp:      OK                     -- pid:  2659,  rss:   1.8 MB,  swap:   0 MB

Information is read from the /proc filesystem or via the ps utility.

Configuration:

The processes to check are configured in /etc/admin-status.conf with one line per process. No comment or blank lines are allowed. Each line contains the process or service name, pid file and optional command-line match:

cron    /var/run/crond.pid
sshd    /var/run/sshd.pid
local   -       local-command

If - is specified as the pid file, the process lookup is made by name (optionally using the command-line match). Process lookup is also made by name if the pid file didn't exist or didn't match a running process.
